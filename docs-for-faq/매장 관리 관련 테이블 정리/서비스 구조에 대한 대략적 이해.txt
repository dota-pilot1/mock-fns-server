■ 이 프로젝트(FSCPS)는 무엇인가?
================================================================================

  CJ프레시웨이 식음(FNB) 통합결제 서비스의 관리자 백오피스(BO)이다.
  매장 관리가 이 프로젝트의 핵심이며,
  매장 정보/스케줄/브랜드/쿠폰 등을 등록·수정·관리하는 것이 주된 역할이다.


■ 두 가지 매장 체계
================================================================================

  FW 체계     = CJ프레시웨이 전체 매장 총괄 (인천, 김포, 김해, 제주 등 전부)
  AIRSTAR 체계 = 인천공항 AIRSTAR 플랫폼에 내보낼 매장 정보

  AIRSTAR는 인천공항이 운영하는 통합 서비스 플랫폼이고,
  매장 정보를 연동하면 공항 이용객이 앱/키오스크 등에서
  매장 검색, 영업시간 확인, 메뉴 조회 등을 할 수 있다.
  다국어(5개국)가 필수인 이유도 외국인 이용객 때문이다.


■ 전체 서비스 구조
================================================================================

  ┌─────────────────────────────┐    ┌──────────────────────────────┐
  │  관리자 BO (이 프로젝트)     │    │  고객 서비스 (별도 서버)      │
  │                             │    │                              │
  │  cjfw-fscps-bo-front        │    │  인천공항 AIRSTAR 앱/키오스크  │
  │  (React)                    │    │  별도 API 서버               │
  │                             │    │  (이 프로젝트 범위 밖)        │
  │  cjfw-fscps-bo-api          │    │                              │
  │  (Spring Boot)              │    │                              │
  └──────────────┬──────────────┘    └──────────────┬───────────────┘
                 │                                  │
                 └──────── 같은 DB 참조 ────────────┘

  - BO에서 매장/스케줄/쿠폰/브랜드 등을 등록·수정하면
  - 고객 서비스가 그 데이터를 읽어서 앱/키오스크에 표시
  - sync_status 컬럼은 두 시스템 간 연동 상태를 추적하기 위한 것


■ 체계별 대표 테이블 — 이름만 봐도 구분 가능
================================================================================

  테이블 네이밍 규칙:
    FW 기존 테이블     → bs_xxx_m_if   (_if 접미사 = 인터페이스/연동 테이블)
    AIRSTAR 테이블     → bs_airstar_*  (airstar가 들어가면 AIRSTAR 체계)
    매핑(브릿지) 테이블 → bs_airstar_store_map

  ┌─────────────┬──────────────────────────────┬─────────────────────┐
  │ 체계        │ 테이블                       │ 역할                │
  ├─────────────┼──────────────────────────────┼─────────────────────┤
  │ FW 기존     │ bs_shop_m_if                 │ 점포 (계약 단위)    │
  │             │ bs_ste_m_if                  │ 사이트 (T1, T2 등)  │
  │             │ bs_crnr_m                    │ 코너 (한식, 양식)   │
  │             │ bs_cust_m_if                 │ 고객사              │
  │             │ bs_emp_m_if                  │ 직원                │
  ├─────────────┼──────────────────────────────┼─────────────────────┤
  │ AIRSTAR     │ bs_airstar_store             │ AIRSTAR 매장        │
  │             │ bs_airstar_store_schedule    │ 일별 영업 스케줄    │
  │             │ bs_airstar_store_img         │ 매장 이미지         │
  │             │ bs_airstar_store_seat        │ 좌석 정보           │
  │             │ bs_airstar_brand             │ 브랜드              │
  ├─────────────┼──────────────────────────────┼─────────────────────┤
  │ 브릿지      │ bs_airstar_store_map         │ FW ↔ AIRSTAR 연결  │
  └─────────────┴──────────────────────────────┴─────────────────────┘

  → 테이블명에 "airstar"가 있으면 AIRSTAR 체계
  → "_m_if"로 끝나면 FW 기존 체계 (ERP 연동 테이블)
  → 이름만으로 거의 바로 구분 가능
