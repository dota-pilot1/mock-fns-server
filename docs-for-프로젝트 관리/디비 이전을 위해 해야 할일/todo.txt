============================================================
  QA DB → mock 로컬 DB 이전 작업 목록
  작성일: 2026-03-14
============================================================

■ 현재 환경
  - mock 로컬 DB: Docker PostgreSQL (localhost:5432/mockfns, user: mockfns)
  - QA DB: AWS Aurora PostgreSQL (fwsafsdbq, user: fscpsapp)
  - 스키마: fscps

■ 이미 완료된 항목
  [V] fscps 스키마 생성
  [V] fscps.bs_airstar_store_schedule 테이블 (매장 스케줄)
  [V] fscps.cm_cl_code_m_if 테이블 (공통코드 분류) — 시드 데이터 21건
  [V] fscps.cm_code_m_if 테이블 (공통코드) — 시드 데이터 92건
  [V] Mapper/Service/Controller 구현 (실무 동일 구조)


============================================================
  1단계: QA DB 덤프 (월요일 — QA DB 접속 가능할 때)
============================================================

■ 공통코드 덤프
  pg_dump -h <qa-host> -p 5432 -U fscpsapp -d fwsafsdbq \
    -t fscps.cm_cl_code_m_if \
    -t fscps.cm_code_m_if \
    --data-only --no-owner --no-privileges \
    > common-code-dump.sql

■ 매장 마스터 관련 테이블 (필요 시)
  pg_dump -h <qa-host> -p 5432 -U fscpsapp -d fwsafsdbq \
    -t fscps.bs_airstar_store_m \
    -t fscps.bs_airstar_store_corner_m \
    -t fscps.bs_airstar_store_schedule \
    --data-only --no-owner --no-privileges \
    > store-dump.sql

■ 메뉴 관련 테이블 (추후 화면 개발 시)
  pg_dump -h <qa-host> -p 5432 -U fscpsapp -d fwsafsdbq \
    -t 'fscps.bs_airstar_menu*' \
    --data-only --no-owner --no-privileges \
    > menu-dump.sql

■ 참고: QA DB 호스트
  - Master: cjfwi-qa-fw-safs-aurorapg.cluster-cvzwl67okbbg.ap-northeast-2.rds.amazonaws.com
  - Read:   cjfwi-qa-fw-safs-aurorapg.cluster-ro-cvzwl67okbbg.ap-northeast-2.rds.amazonaws.com


============================================================
  2단계: 로컬 mock DB에 import
============================================================

■ 기존 시드 데이터 정리 (QA 덤프로 대체)
  docker exec -i mockfns-db psql -U mockfns -d mockfns -c "
    TRUNCATE fscps.cm_cl_code_m_if CASCADE;
    TRUNCATE fscps.cm_code_m_if CASCADE;
  "

■ 덤프 파일 import
  docker exec -i mockfns-db psql -U mockfns -d mockfns < common-code-dump.sql
  docker exec -i mockfns-db psql -U mockfns -d mockfns < store-dump.sql

■ 데이터 확인
  docker exec mockfns-db psql -U mockfns -d mockfns -c "
    SELECT comm_cl_cd, COUNT(*) FROM fscps.cm_code_m_if
    WHERE use_yn='Y' GROUP BY comm_cl_cd ORDER BY comm_cl_cd;
  "


============================================================
  3단계: 누락 테이블 확인 및 추가 생성
============================================================

■ QA DB에서 fscps 스키마 테이블 목록 조회
  psql -h <qa-host> -U fscpsapp -d fwsafsdbq -c "
    SELECT table_name FROM information_schema.tables
    WHERE table_schema = 'fscps' ORDER BY table_name;
  "

■ 로컬에 없는 테이블 → DDL 덤프
  pg_dump -h <qa-host> -p 5432 -U fscpsapp -d fwsafsdbq \
    -n fscps --schema-only --no-owner --no-privileges \
    > fscps-schema-ddl.sql

  # 필요한 테이블만 선별해서 로컬에 실행
  docker exec -i mockfns-db psql -U mockfns -d mockfns < fscps-schema-ddl.sql


============================================================
  4단계: 검증
============================================================

□ mock 서버 재시작 후 프론트엔드 로그인 → 공통코드 로딩 확인
□ 매장 스케줄 관리 화면 → 조회/저장 정상 동작 확인

============================================================
  주의사항
============================================================

- pg_dump 시 --no-owner --no-privileges 필수 (로컬 user가 다르므로)
- QA DB 비밀번호는 암호화되어 있음 (ENC(...)) → 복호화 필요하거나 직접 접속
- 덤프 파일에 스키마(fscps.) 접두사가 포함되므로 로컬에도 fscps 스키마 필요 (이미 생성됨)
- 대용량 테이블은 WHERE 조건으로 필요한 데이터만 덤프 권장
